# Screenshot Guide

Use this folder for final portfolio screenshots before publishing the repository or portfolio case study.

Do not commit broken image references. Add screenshot files first, then reference them from this guide or the main README.

## Recommended Screenshots

| Area | Suggested file name | Capture notes |
|---|---|---|
| Home page | `home-page.png` | Show the first viewport with the main storefront entry point and navigation. |
| Product catalog | `product-catalog.png` | Show products, search/filter controls, and at least one visible product card. |
| Product details | `product-details.png` | Show product image area, description, price, stock state, and add-to-cart action. |
| Cart | `cart.png` | Show cart items, quantity controls, total, clear-cart action, and checkout action. |
| Checkout | `checkout.png` | Show the checkout summary and simulated payment language if visible. |
| My orders | `my-orders.png` | Show order history with order status and payment status. |
| Admin dashboard | `admin-dashboard.png` | Show KPI cards or dashboard metrics with admin navigation. |
| Admin products | `admin-products.png` | Show product management table/form controls. |
| Admin orders | `admin-orders.png` | Show admin order list and status management controls. |

## Capture Checklist

- Use seeded demo data so the catalog and admin dashboard look populated.
- Avoid showing real secrets, real customer data, private tokens, or local `.env` values.
- Keep payment wording honest: this project uses simulated payments only.
- Prefer consistent browser width across screenshots, for example 1440px desktop and one optional mobile capture.
- Crop browser chrome only if the image still clearly represents the actual application screen.
